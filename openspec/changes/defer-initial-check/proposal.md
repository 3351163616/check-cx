# Change: 推迟首次检测至第一个轮询周期

## Why

当前行为：应用启动时轮询器初始化会立即执行一次检测（`tick()`），导致第一次访问页面时触发即时检测。

问题：
1. 用户可能在服务刚启动、环境尚未稳定时访问，立即检测可能产生误报
2. 如果启动时 API 密钥或网络配置尚未就绪，首次检测会失败并记录错误历史
3. 与 "定时轮询" 的语义不一致——既然设定了轮询间隔，首次检测也应遵循间隔

期望行为：启动后等待第一个轮询周期（`POLL_INTERVAL_MS`）到达后再执行首次检测。

## What Changes

- 移除 `lib/core/poller.ts` 中启动时的立即 `tick()` 调用
- 仅保留 `setInterval` 定时器，首次检测在第一个间隔到达后执行
- 更新日志输出，明确提示首次检测将在 N 秒后执行

## Impact

- Affected code: `lib/core/poller.ts:114`
- 用户体验：首次访问页面时可能看到空数据，直到第一个轮询周期完成
- 数据一致性：所有检测记录都来自定时轮询，没有特殊的"启动检测"
